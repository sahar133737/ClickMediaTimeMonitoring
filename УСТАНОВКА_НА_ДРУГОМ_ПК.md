# Запуск ClickMediaWorkTime на другом компьютере

## Требования

- Windows 10/11
- [.NET Framework 4.8](https://dotnet.microsoft.com/download/dotnet-framework/net48)
- SQL Server **или** LocalDB (идёт с Visual Studio; для демо можно установить [SQL Server Express](https://www.microsoft.com/sql-server/sql-server-downloads))

## Что копировать

Скопируйте **всю папку** сборки, например:

`ZAIKA DIPLOMA\bin\Release\`

В ней должны быть:

- `ClickMediaWorkTime.exe`
- `ClickMediaWorkTime.exe.config` — **обязательно** (настройки БД)
- папка `Sql\` с файлами `01_create_schema.sql`, `02_seed_demo_data.sql`

Не копируйте только `.exe` без `.config` и без `Sql\`.

## Настройка базы данных

1. Откройте `ClickMediaWorkTime.exe.config` в блокноте.
2. В обоих параметрах `MasterConnection` и `AppConnection` измените **Server** на свой экземпляр:

| Вариант | Server |
|--------|--------|
| LocalDB | `(localdb)\MSSQLLocalDB` |
| SQL Express | `.\SQLEXPRESS` |
| Именованный экземпляр | `ИМЯ_КОМПЬЮТЕРА\ИМЯ_ЭКЗЕМПЛЯРА` |

3. Сохраните файл и запустите `ClickMediaWorkTime.exe`.

При первом запуске программа сама создаст базу `ClickMediaTimeDB` (скрипт из папки `Sql`).

## Ошибка про сертификат SQL Server

Если появляется сообщение о недоверенном сертификате цепочки TLS, в проекте это уже учтено: в код добавляются `TrustServerCertificate=True` и `Encrypt=False`. Убедитесь, что запускается **новая** сборка после обновления.

## Ошибка про сертификат приложения / ClickOnce

Сборка для переноса — обычный **Release** из Visual Studio (`bin\Release`), не установщик ClickOnce. Подпись манифеста в проекте отключена специально для запуска на чужих ПК.
